# SignTeb AI Web Chat — ویجت چت هوشمند پزشکی (مستقل و سفیدبرچسب)

افزونه‌ی وردپرسی **کاملاً مستقل** برای افزودن یک ویجت چت هوشمند پزشکی به هر سایت کلینیک/پزشک. این افزونه به هیچ افزونه‌ی دیگری (از جمله SignTeb Medical Core یا SignTeb AI Connect) وابسته نیست؛ تمام اطلاعات کلینیک را از پنل تنظیمات خودش می‌خواند و روی هر وردپرس تازه‌نصب با هر تمی (حتی Twenty Twenty-Four) بلافاصله کار می‌کند.

> این مستند فرض می‌کند خواننده هیچ پیش‌زمینه‌ای از اکوسیستم SignTeb ندارد.

---

## ویژگی‌ها

- **دو سرویس‌دهنده‌ی هوش مصنوعی، قابل انتخاب:** Anthropic Claude یا OpenAI. در تنظیمات یک Dropdown انتخاب می‌کنید و فیلد کلید + مدلِ همان سرویس بلافاصله (با JS، بدون رفرش) نمایش داده می‌شود. مدل‌ها هاردکد نیستند و قابل تغییر دستی‌اند. پیش‌فرض‌ها: Anthropic → `claude-haiku-4-5-20251001` (سبک و کم‌هزینه)، OpenAI → `gpt-4o-mini`.
- **کلید API رمزنگاری‌شده:** کلید هر دو سرویس به‌صورت AES-256 رمزنگاری‌شده در دیتابیس ذخیره می‌شود (نه plaintext).
- **حالت محتوای دستی:** نام کلینیک، تخصص، خدمات و قیمت‌ها، آدرس، تلفن، واتس‌اپ، ساعات کاری و لینک رزرو — همه دستی وارد می‌شوند و در هر درخواست به System Prompt تزریق می‌شوند.
- **لایه‌ی ایمنی پزشکی (غیرقابل‌حذف):** هرگز تشخیص قطعی/تجویز دارو نمی‌دهد؛ در پاسخ‌های مرتبط با علائم، دیسکلیمر اضافه می‌کند؛ در موارد اورژانسی فوراً شماره‌ی اورژانس را نشان داده و مکالمه‌ی AI را متوقف می‌کند. این چک هم در System Prompt و هم در یک لایه‌ی post-processing انجام می‌شود.
- **ویجت سفیدبرچسب (White-label):** رنگ اصلی/ثانویه، نام چت‌بات، آواتار، پیام‌ها، فوتر و جهت (RTL/LTR) همه قابل تغییر کامل‌اند — بدون هیچ برندینگ هاردکد.
- **بدون jQuery:** Vanilla JS خالص، با افکت تایپ طبیعی، دکمه‌های پاسخ سریع، و کارت CTA درون‌خطی (واتس‌اپ/تماس/رزرو) هنگام تشخیص نیت رزرو.
- **بارگذاری مشروط asset:** JS/CSS فقط وقتی ویجت واقعاً رندر می‌شود لود می‌گردد.
- **تاریخچه‌ی مکالمات + آمار:** مشخص‌بودن اینکه کدام مکالمه به CTA تبدیل شده (اثبات ROI)، نرخ تبدیل، و پرتکرارترین پرسش‌ها (ورودی استراتژی سئو).
- **REST + admin-ajax:** اگر هاست REST API را محدود کند، خودکار به admin-ajax سوییچ می‌کند.
- **حالت آزمایشی + لایسنس:** ۵۰ پیام رایگان، سپس نیاز به فعال‌سازی کلید لایسنس سالانه (معماری قابل گسترش به سرور لایسنس).
- **چندزبانه:** فارسی (پیش‌فرض)، عربی و انگلیسی؛ تشخیص خودکار زبان از روی پیام کاربر. فایل `.pot` برای ترجمه آماده است.

## نصب

1. پوشه‌ی `signteb-web-chat` را به `wp-content/plugins/` کپی کنید (یا فایل ZIP را از مسیر «افزونه‌ها ← افزودن ← بارگذاری افزونه» آپلود کنید).
2. افزونه را فعال کنید. در فعال‌سازی، جدول‌های `{prefix}swc_conversations` و `{prefix}swc_messages` ساخته می‌شوند.
3. به منوی **SignTeb Chat** در پیشخوان بروید.

## پیکربندی

تنظیمات در یک صفحه با تب‌های زیر سازمان‌دهی شده‌اند:

| تب | کار |
|---|---|
| **هوش مصنوعی** | انتخاب سرویس‌دهنده، وارد کردن کلید API، انتخاب مدل، لحن، زبان، محدودیت نرخ |
| **اطلاعات کلینیک** | نام، تخصص، تماس، ساعات کاری، شماره اورژانس، لینک رزرو، فهرست خدمات و قیمت‌ها |
| **ظاهر ویجت** | رنگ‌ها، نام چت‌بات، آواتار، جهت، فونت، فوتر، پیام خوش‌آمد، دکمه‌های سریع |
| **تاریخچه مکالمات** | فهرست مکالمات، فیلتر «فقط لیدها»، مشاهده‌ی متن کامل هر مکالمه |
| **آمار** | تعداد مکالمه، نرخ تبدیل CTA، پرتکرارترین پرسش‌ها |
| **لایسنس** | وضعیت نسخه آزمایشی و فعال‌سازی کلید |

### فهرست خدمات
هر خط یک خدمت، با فرمت `نام | قیمت`. مثال:

```
ویزیت عمومی | ۲۵۰ هزار تومان
لیزر موهای زائد | از ۵۰۰ هزار تومان
```

### فونت Vazirmatn (اختیاری)
برای فونت باندل‌شده، فایل `vazirmatn.woff2` را در `assets/fonts/` قرار دهید و گزینه‌ی «فونت باندل‌شده» را فعال کنید. اگر فایل موجود نباشد، افزونه به‌صورت ایمن به فونت سیستمی (Vazirmatn/Tahoma) برمی‌گردد و هیچ خطای 404 نمی‌دهد.

## معماری (برای توسعه‌دهنده)

- **PSR-4-style autoloading:** یک کلاس = یک فایل، با نام‌گذاری `SWC_Foo_Bar` → `class-foo-bar.php`. نگاشت کلاس‌به‌فایل به‌صورت صریح در `includes/class-autoloader.php` تعریف شده تا هرگز یک `require` به فایلی که ساخته نشده اشاره نکند.
- **Interface-driven AI layer:** افزودن سرویس‌دهنده‌ی جدید فقط نیازمند یک کلاس پیاده‌سازی‌کننده‌ی `SWC_AI_Provider_Interface` است.
- **Repository pattern** برای دیتابیس (نه Active Record).
- **محافظت خروجی JSON** با `ob_start()` + `register_shutdown_function` (برای هاست‌هایی که notice چاپ می‌کنند).
- **فراخوانی API** با `wp_remote_post` و timeout حداکثر ۲۲ ثانیه (سازگار با `max_execution_time: 30` هاست‌های ایرانی) و مدیریت کامل خطا — هرگز کرش نمی‌کند.

### Hookها
- اکشن‌ها: `swc_lead_detected`, `swc_message_handled`, `swc_license_status_changed`
- فیلترها: `swc_should_render`, `swc_system_prompt`, `swc_license_is_active`, `swc_trial_limit`

### REST
`POST /wp-json/signteb-web-chat/v1/message` (nonce-protected, rate-limited). در صورت محدودیت REST، fallback به `admin-ajax.php?action=swc_chat_message`.

## حذف
با حذف افزونه از پیشخوان، `uninstall.php` جدول‌ها و همه‌ی optionها را پاک می‌کند. صرفِ **غیرفعال‌سازی** هیچ داده‌ای را حذف نمی‌کند.

## امنیت
nonce و sanitize/escape روی همه‌ی ورودی‌ها اعمال می‌شود. این افزونه برای نصب روی سایت‌های ناشناس طراحی شده است؛ هیچ `exec()`/`proc_open()` استفاده نمی‌شود.
